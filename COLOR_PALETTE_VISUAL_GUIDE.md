## Accessibility Considerations

### For Color Blind Users
- All UI information is not conveyed by color alone
- Text labels accompany all colored elements
- Patterns/icons supplement color coding
- Sufficient contrast compensates for color confusion

### For Low Vision Users
- Large text sizes
- High contrast ratios
- Clear focus indicators
- Adequate spacing between elements

### For Motor Impairments
- Button sizes adequate for clicking (minimum 44x44px)
- Clear focus states for keyboard navigation
- No time-limited color changes
- Accessible form controls

## Implementation Notes

### CSS Variables Location
All color values are defined in: `src/app.css`

```css
@theme {
  --color-primary-DEFAULT: #dd4444;
  --color-primary-dark: #cc3333;
  --color-primary-light: #ffe0de;

  --color-secondary-DEFAULT: #9370db;
  --color-secondary-dark: #7c3aed;
  --color-secondary-light: #ede9fe;

  --color-accent-DEFAULT: #2ba89b;
  --color-accent-dark: #1f9a8a;
  --color-accent-light: #d4f4f2;

  --color-sunshine-DEFAULT: #d4a500;
  --color-sunshine-dark: #b89100;
  --color-sunshine-light: #fff9e6;

  --color-nature-DEFAULT: #4ba85f;
  --color-nature-dark: #3a9349;
  --color-nature-light: #e8f8eb;
}
```

### Component Usage

Colors are applied through:
1. Tailwind classes (e.g., `bg-primary`, `text-primary`)
2. CSS custom properties (e.g., `var(--color-primary-DEFAULT)`)
3. Dynamic class binding (e.g., `class:bg-primary-light`)

### Maintenance Guidelines

When updating colors:
1. Update CSS variables in `src/app.css`
2. Verify contrast ratios meet WCAG AA minimum
3. Test in multiple browsers
4. Verify on light and dark backgrounds
5. Check colorblind simulation tools
